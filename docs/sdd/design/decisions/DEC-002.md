# DEC-002: Environment Variables for Depth Propagation

## Status
Approved

## Context
Need a mechanism to propagate orchestration context (depth, session ID, channels) to child panes.

## Decision
Use environment variables: `CCORCH_DEPTH`, `CCORCH_SESSION_ID`, `CCORCH_PARENT_CHANNEL`, `CCORCH_WORK_DIR`, `CCORCH_TIMEOUT`, `CCORCH_MAX_PANES`.

## Rationale
- **Simple**: No config files or IPC mechanisms needed
- **Inherited by subprocesses**: Set once at pane creation, available to all child processes
- **CLI flag integration**: `--append-system-prompt` can reference env vars
- **Tamper-resistant for depth control**: Combined with `--disallowedTools "Agent"` at DEPTH=3, the structural enforcement doesn't rely on the AI respecting a prompt instruction

## Risks
- Environment variable names could collide with other tools → mitigated by `CCORCH_` prefix
- Values visible in `/proc/<pid>/environ` → acceptable since these are non-secret metadata
