# DEC-003: /tmp/ for Result Storage

## Status
Approved

## Context
Need a location to store orchestration result files exchanged between parent and child panes.

## Decision
Use `/tmp/ccorch/<session_id>/` for all result files.

## Rationale
- **Ephemeral by nature**: Orchestration results are intermediate artifacts, not long-term records
- **No git pollution**: Avoids cluttering the repository with `.gitignore` entries
- **Cross-repository**: Works regardless of which repository the orchestration operates on
- **Session isolation**: Each session gets its own directory via unique timestamp ID
- **ccmemo handles persistence**: Important discoveries are persisted via `record-knowledge`, not raw result files

## Risks
- Files lost on reboot → acceptable since orchestration is a single-session activity
- `/tmp/` cleanup by OS cron → mitigated by session completing within hours, not days

## Cleanup
- Result files are not automatically cleaned up (OS `/tmp/` cleanup handles this)
- Future enhancement: add a cleanup command or TTL-based cleanup
