# DEC-001: tmux wait-for for Signaling

## Status
Approved

## Context
Need a mechanism for child panes to notify parent panes of task completion. Options considered:
1. Polling via flag files
2. `tmux send-keys` to parent
3. `tmux wait-for` signaling

## Decision
Use `tmux wait-for` as the signaling primitive.

## Rationale
- **Blocking primitive**: `tmux wait-for` blocks until signaled — no polling overhead
- **Combined with `run_in_background`**: Converts blocking wait into async notification, letting the parent continue working
- **No terminal input**: Unlike `send-keys`, doesn't inject text into the parent's terminal
- **Built-in to tmux**: No additional dependencies (available since tmux 1.8)
- **Reference implementation**: Validated by the Zenn article (https://zenn.dev/genda_jp/articles/cfcaeeb51d23e7)

## Risks
- If child fails to signal, parent blocks forever → mitigated by timeout watchdog
- tmux session disconnect breaks signal flow → mitigated by `trap EXIT`

## Alternatives Rejected
- **Flag file polling**: Simple but wasteful (CPU cycles for polling) and timing-dependent
- **send-keys**: Injects text into terminal, fragile if parent is mid-output
