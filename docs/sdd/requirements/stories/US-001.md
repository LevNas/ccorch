# US-001: Orchestration Execution

## User Story

**As a** Claude Code user
**I want to** run the `/ccor` skill to automatically decompose and parallelize complex tasks
**So that** I can efficiently handle large-scale tasks without manually creating and managing tmux panes

## Acceptance Criteria (EARS Notation)

### Functional Requirements

- **REQ-001-001**: When the user runs `/ccor`, the system shall generate a session ID (UNIX timestamp) and create the `/tmp/ccorch/<session_id>/` directory
- **REQ-001-002**: When the user runs `/ccor`, the system shall create a Main Brain pane in a new tmux window and launch Claude Code with `CCORCH_DEPTH=1`
- **REQ-001-003**: After Main Brain pane launch, the user's session shall continue parallel work while waiting for completion via `run_in_background: true`
- **REQ-001-004**: When the Main Brain completes, the system shall present a completion notification and result summary to the user
- **REQ-001-005**: The Main Brain shall autonomously analyze the received task and decide on decomposition into child tasks
