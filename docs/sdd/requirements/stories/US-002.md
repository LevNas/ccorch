# US-002: 3-Level Depth Control

## User Story

**As a** Claude Code user
**I want to** orchestration depth to be limited to 3 levels
**So that** I can prevent resource exhaustion and chaotic states caused by unlimited pane proliferation

## Acceptance Criteria (EARS Notation)

### Functional Requirements

- **REQ-002-001**: When the Main Brain (DEPTH=1) creates a child pane, the system shall set `CCORCH_DEPTH=2` on the child pane
- **REQ-002-002**: When a Child (DEPTH=2) creates a grandchild pane, the system shall set `CCORCH_DEPTH=3` on the grandchild pane
- **REQ-002-003**: If `CCORCH_DEPTH=3` (Grandchild), the system shall reject pane creation requests and return an error message
- **REQ-002-004**: When creating each pane, the system shall propagate `CCORCH_SESSION_ID`, `CCORCH_PARENT_CHANNEL`, and `CCORCH_WORK_DIR` as environment variables

## Environment Variables

| Variable | Purpose | Example |
|----------|---------|---------|
| `CCORCH_DEPTH` | Current depth level | `1`, `2`, `3` |
| `CCORCH_SESSION_ID` | Unique session identifier | `1711871234` |
| `CCORCH_PARENT_CHANNEL` | Notification channel to parent | `CCORCH_DONE_1711871234_child1` |
| `CCORCH_WORK_DIR` | Result storage directory | `/tmp/ccorch/1711871234/` |
