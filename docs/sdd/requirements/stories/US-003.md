# US-003: tmux wait-for Communication

## User Story

**As a** Claude Code user
**I want to** completion notifications between parent and child panes to be reliably delivered via signals
**So that** I get more efficient and precisely timed completion detection than polling

## Acceptance Criteria (EARS Notation)

### Functional Requirements

- **REQ-003-001**: When a child pane completes its task, the system shall write results to a file in `CCORCH_WORK_DIR` and send a signal via `tmux wait-for -S <CCORCH_PARENT_CHANNEL>`
- **REQ-003-002**: The parent pane shall execute `tmux wait-for <channel>` with `run_in_background: true` and read the result file upon signal reception
- **REQ-003-003**: If a child pane does not complete within the timeout period (default: 600 seconds), the system shall send a timeout notification to the parent and terminate the pane
- **REQ-003-004**: When a child pane terminates abnormally, the system shall record the error state in the result file and send a signal to the parent

## Data Flow

```
Child pane:
  1. Execute task
  2. Write results to /tmp/ccorch/<id>/child-<n>.md
  3. tmux wait-for -S CCORCH_DONE_<id>_child<n>

Parent pane:
  1. run_in_background: tmux wait-for CCORCH_DONE_<id>_child<n>
  2. (Waits in background, continues own work)
  3. Signal received → Read result file
```

## Result File Format

```markdown
---
status: success | error | timeout
depth: 2
task: "Task summary"
started: 2026-03-31T16:30:00+09:00
completed: 2026-03-31T16:35:00+09:00
---

# Results

## Execution Summary
...

## Changed Files
...

## Discoveries
...
```

## Notes

- Timeout value is configurable via `CCORCH_TIMEOUT` environment variable
- Timeout implementation: `(sleep $TIMEOUT && tmux wait-for -S <channel>) &` pattern
- A wrapper script is needed to execute `tmux wait-for -S` as the last command in child panes
